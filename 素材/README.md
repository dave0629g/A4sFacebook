# 素材區 ── 小編交件 → 維護者一鍵更新成果

**誰操作這裡**：維護這個 repo 的人（＝幫小編的「搭檔」，可能就是你）。**小編不用碰這個資料夾、不用改 yaml、不用跑程式**——小編只負責做 FB 工作、把值與檔案交給你；你填進來、跑一行指令，就生出可直接貼的成品。小編該做什麼看 `執行包/10_小編任務清單.md`，跟小編收什麼看 `執行包/09_素材索取清單.md`。

## 三步驟工作流（維護者操作）

```
①  把小編交來的「值」填進  素材/提供資料.yaml      （數字、連結、曲名、得獎名單、聽後感…）
    把小編交來的「檔案」丟進 素材/收件匣/            （截圖、圖卡…；照片走原圖來源，見下）
②  到專案根目錄執行：  python3 素材/update.py
③  立即產生／更新：
      素材/成品/        ← 已代入的貼文、私訊、LINE 訊息，可直接複製給小編貼出
      素材/狀態.md      ← 完成度、🚨現在該處理、下一個死線
      素材/實品/        ← 收件匣的檔案自動歸位（不進 git）
```

沒填的值，成品會原樣保留 `【佔位】`，一眼看出還缺什麼，**程式不會瞎掰**。

> 注意：另有一套「自動化程式」（07_自動化程式.gs，Google Apps Script 選單 ①–⑥）是裝在小編的試算表裡、由**小編**操作的，跟這裡的 `update.py` 是兩回事，別混淆。

## 小編 Checklist 網頁

給很忙、眼力不好的小編：大字、高對比、做完點一下方框就好，點擊即時記錄。
跑：`streamlit run 素材/小編checklist.py`。要給遠端小編開、進度怎麼流回 `狀態.md`，見 `小編checklist_使用說明.md`。

## 照片放哪？（重要）

照片**不進這個 repo**。目前原圖在 `Projects/A4s/Line相簿/`（74 張；那個 repo 已 gitignore，不會外流）；備用位置是 repo 外的 `~/A4s音樂會素材原圖/`。

`update.py` 最上面的 `PHOTO_SOURCES` 列出所有來源，程式只**清點張數**寫進 `狀態.md`，**不複製照片**。哪些進第一/二波是審片時人工挑。要再加來源資料夾就改 `PHOTO_SOURCES`。詳見 `LINE相簿同步教學.md`。最高畫質請用 Google Drive 共用資料夾，LINE 當備援。

## 個資紅線（務必遵守）

本 repo 是公開的。下列東西**只存共用資料夾試算表，絕不進這個 repo、也不要填進 yaml**：

- 填答者／得獎者的真實姓名、電話、Email、身分證、戶籍地址、稅務資料
- 抽獎名單的「暱稱＋時間戳＋邀請團員」對應表（領獎三重比對用）
- 領獎合影、刊登同意私訊截圖（含未遮蔽姓名）、抽獎全程錄影

yaml 裡只放「本來就要公開」的值：遮蔽版得獎名單（王Ｏ明）、曲名、參加人數、已同意刊登的聽後感、KPI 數字。

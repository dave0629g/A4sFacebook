> 這個資料夾在 git 專案【外面】，永遠不會上傳，可以安心放未遮蔽的原圖。

## 存到哪裡（你問的就是這個）

| LINE 相簿內容 | 存到這個資料夾 |
|---|---|
| 演出、謝幕照（第一波） | `~/A4s音樂會素材原圖/第一波_演出謝幕/` |

`~` 是你的家目錄，完整路徑是 `/Users/dave0629/A4s音樂會素材原圖/`。在 Finder 用 `Cmd+Shift+G` 貼路徑即可跳過去；資料夾已經建好了。

**為什麼不放專案資料夾？** 因為專案是 git 版控、會上傳 GitHub。照片放外面這個資料夾，從根本上不可能被 git 碰到——這也是你交代「只在本地硬碟、不要放到 git」的做法。

## LINE 電腦版下載步驟

LINE 沒有開放給程式自動抓相簿的官方介面，所以用「電腦版手動批次下載」最穩：

1. 開 **LINE 電腦版**（Mac App）→ 進入該群組聊天室
2. 右上角 **「≡」選單 → 相簿**（或聊天室資訊裡的「相簿」）
3. 點進要下載的相簿
4. 右上 **「⋯」→「全部儲存」**；或勾選多張照片後按 **「儲存」/「下載」**
5. 跳出選資料夾視窗時，**選到上表對應的子資料夾**
6. 下載完成

> ⚠️ LINE 相簿會**壓縮畫質**。要最高畫質的原檔，請改用 **Google Drive 共用資料夾**上傳原圖，LINE 這條當備援。

## 下載完做什麼

到專案根目錄跑一次：

```bash
python3 素材/update.py
```

程式會**清點**這兩個資料夾各有幾張、寫進 `素材/狀態.md`，但**不會把照片複製進 git**。照片始終只在你本機。

## 想要「下載完自動清點」（選用）

如果懶得每次手動跑，可開一個背景監看，照片一進資料夾就自動重跑 update.py。需要先裝 `fswatch`（`brew install fswatch`），然後：

```bash
fswatch -o ~/A4s音樂會素材原圖 | while read; do python3 素材/update.py; done
```

開著這個終端機視窗，之後每次 LINE 存圖進去，狀態就自動更新。不需要時關掉視窗即可。

## 常見問題

- **找不到「相簿」**：照片若是直接傳在聊天室而非建成相簿，要先在聊天室把它們「加入相簿」，或長按單張照片逐張儲存。
- **手機版能不能直接同步到電腦？** 不行，手機版沒有批次存到電腦的功能；用電腦版，或手機存到自己相簿後再用 AirDrop / iCloud 照片傳到電腦這個資料夾。
- **照片會不會不小心上傳 GitHub？** 不會。它們在 `~/A4s音樂會素材原圖/`，在專案資料夾之外，git 看不到；就算放進專案，`.gitignore` 也擋掉了非佔位圖。

# 08 自動化使用說明

> 一支 Google Apps Script，把活動中「機器做得來的事」全部接管。**完全免費**，跑在 Google 自己的環境，不用裝任何東西。
> 程式碼在 `07_自動化程式.gs`。

## 程式接管了哪些工作

| 原本（手動） | 現在（程式） |
|---|---|
| 照 01 逐字建表單 10 題＋條款（60–90 分） | 選單點「① 建立表單」，30 秒建好（含自動設定「每欄僅限一則回應」與「發布」） |
| 表單排程關閉（怕介面找不到選項） | 「② 設定自動關閉」，時間到自動關＋顯示截止訊息 |
| 6/18 人工撈「誰已填」再分別私訊團員 | 「③ 更新進度報表」：各團員的已填名單一鍵列好，直接複製私訊 |
| 6/26 手動排序去重、剔無效票、算票（40 分） | 「④ 截止後整理＋計票」：去重、無效票、利益迴避、計票排名、摘要一鍵完成 |
| 手動編號做轉盤名單 | 「⑤」一鍵產生 wheelofnames 名單＋備用亂數抽獎 |
| 結案逐欄刪個資、抄 Email 名單、刪表單端回覆 | 「⑥」一鍵轉存 Email＋清除個資＋刪表單端回覆 |

**程式不做（也不該做）的**：發私訊、FB 發文（用粉專內建排程）、追蹤者查核、照片審片。

## 跟其他檔案怎麼分工（先讀這段再開跑）

- **00 時程照舊是總指揮**，只有一處改變：建置日的「照 01 建表單（90 分）」改成「照本文件安裝＋①②（約 25 分）」。
- **01 仍要做**：步驟 0（活動辦法文件＋辦法短網址，**按①之前就要做好**，CONFIG 要填）、文末的「取得回覆者連結 → 縮 A/B 兩條短網址 → 手機實測」。中間的逐題建表全部跳過（01 保留作為手動備案與題目對照）。
- **04 仍要做**：第二節的轉盤操作（2 位幹部見證、全程錄影、每輪先截圖再按 Remove）、第三～五節（開獎貼文、三重比對領獎查核、稅務、結案個資時程）。04 第一節「名單整理」與第二節末的亂數備案已由④⑤取代，**不要再手動做一遍**。

## 安裝（小編做一次，約 10 分鐘）

1. 到 sheets.google.com 建一個**新的空白試算表**，命名「票選活動_回覆與管理」
2. 上方選單「擴充功能」→「Apps Script」
3. 把編輯器裡原有的內容全部刪掉，將 `07_自動化程式.gs` 全文貼上
4. **改最上面的 CONFIG 設定區**：音樂會名稱、曲目清單（照節目單順序）、團員名單、辦法短網址（**先照 01 的步驟 0 做出辦法文件＋短網址才有得填**）；CLOSE_AT 順手核對一次。其他不用動。改的時候三個鐵則：
   - 只改『』裡的字，**兩側的半形引號 `'` 和行尾的半形逗號 `,` 都要留著**，不要打成中文的「」或，
   - 曲目幾首就留幾行：多的整行刪掉，不夠就複製一行照改，最後一行結尾留著逗號沒關係
   - 曲名或人名裡有英文撇號（如 Can't Help Falling in Love）會讓程式壞掉：把那一行的外框引號改成**雙引號**，寫成 `"Can't Help Falling in Love",`
   - 若整個時程平移：CLOSE_AT、CLOSE_TEXT、ANNOUNCE_TEXT **三個要一起改**
5. 點 💾 儲存 → 回到試算表、**重新整理網頁** → 上方會出現「🎵 票選活動」選單。**如果選單沒出現**，九成是 CONFIG 的引號或逗號改壞了：回「擴充功能→Apps Script」，編輯器會在出錯那一行標紅，照上面鐵則修好、儲存、再重新整理一次
6. 第一次點任何功能會跳出「需要授權」視窗 → 按「確定」→ 選自己的帳號 → 出現「**Google 尚未驗證這個應用程式**」警告頁（**正常！** 自己貼的程式都會這樣）→ 點左下角小字「**進階**」→ 點「**前往 ○○（不安全）**」——「不安全」是 Google 給所有自寫程式的統一標籤，不是程式有問題 → 按「**允許**」。授權完成後**剛剛點的功能不會自動執行，請回到試算表再點一次選單**。程式只動你自己這份試算表和它建立的表單

## 使用時點

| 時間 | 動作 |
|---|---|
| 建置日（00 的 6/12 晚） | 先做 01 步驟 0（辦法文件）→ 安裝本程式 → 點「**① 建立表單**」（表單檔會出現在「我的雲端硬碟」根目錄，可自行拖進活動資料夾，不影響運作）→ 照彈窗確認表單可作答 → 照 01 文末縮 A/B 短網址＋手機實測 → 點「**② 設定自動關閉**」 |
| 活動期間 | 隨時點「**③ 更新進度報表**」看「進度」分頁；6/18 提醒日把各團員那一列的已填名單分別私訊給他們 |
| 6/26 | 先到「**排除名單**」分頁填入：測試回覆的暱稱（類型寫「測試」）、團員家屬的暱稱（類型寫「團員家屬」）→ 點「**④ 截止後整理＋計票**」→ 檢查「整理摘要」「需人工判定」「計票結果」→ 點「**⑤**」拿「轉盤名單」去 wheelofnames 公開抽 → **轉盤操作細節照 04 第二節**（2 位幹部見證、全程錄影、每輪先截圖再 Remove） |
| 6/27–29 | 領獎查核照 04 SOP（人工）；「聽後感_可刊登」分頁就是結案貼文可引用的留言 |
| 7/1 | 先抄存中獎者領獎／稅務資料、確認結案貼文素材已取用 → 點「**⑥ 結案**」（程式會一併刪除表單端回覆，不必再去 forms.google.com 手動刪） |

## 重要限制（誠實條款）

1. 「每欄僅限一則回應」已由程式自動設定；「需人工判定」分頁仍保留作為保險網，正常情況應為空
2. **建好的表單請勿手動改題目文字或 CONFIG 曲目**——程式靠題目標題對應資料欄位，改了會直接報錯擋下（不會給錯資料）。真要重建：刪 FORM_ID 指令碼屬性＋刪舊「表單回應」分頁＋丟棄舊表單，三件都要做（①的提示視窗有完整步驟）
3. 自動關閉靠 Google 觸發器，官方不保證分鐘級精度，可能延遲數分鐘（最長約 15 分鐘；**只會晚關、不會提早關**）——幹部仍設一個 21:00 鬧鐘核對
4. 「備用抽獎結果」每按一次重抽一次，正式抽獎以 wheelofnames 公開過程為準
5. 試算表共用給幹部時給「編輯者」即可；**不要**共用給全體團員（裡面有填答者個資）
6. ③④⑤每按一次都會**整份重寫**對應分頁——人工補進「抽獎池」「計票結果」的資料，再按一次④就會被洗掉。順序務必：按④ → 人工判定補完 → 按⑤，之後不要回頭重按④
7. ⑥清除個資後，試算表的「檔案→版本紀錄」仍留有舊內容。好處：真的忘了先抄中獎者資料，可從版本紀錄救回。壞處：個資並未真正消失——結案後請用「檔案→建立副本」做一份乾淨副本留存統計（副本不帶版本紀錄），再把原始試算表整份刪除並清空垃圾桶，個資才算清乾淨

## KPI 對照（覆盤時從哪裡拿數字）

- 參加人數：「整理摘要」的【參加人數】列（直接填進貼文 4、5；含「需人工判定」筆數，認定剔除後以修正後數字為準）
- 自我聲明追蹤率、Email 訂閱數：「進度」分頁
- 私訊發送數：認領名單試算表打勾欄（這份在團員群組，不在本表）
- 短網址點擊：reurl 後台；追蹤者數：Meta 洞察報告（皆手動）
